# Admin Navigation & Charts Refresh

## Navigation Redesign
- The admin sidebar was completely overhauled to a premium **#0b0b0b** black background with white text.
- Active states now use an elevated dark styling (`#1a1a1a`) with a white left border indicator.
- Disabled "Soon" items no longer use ugly pill chips; they use clean, muted typography (`text-[9px] text-[#6b7280]`) and are non-clickable (`pointer-events-none`).
- Mobile navigation drawer continues to function as expected with the new dark styling.
- Admin loading state properly uses the global `AppLoadingScreen` via the `forceActive` prop.

## Chart Integration (Phase 1)
- **Recharts** was adopted for this phase because it is highly compatible with React/Vite and provides a clean, modern aesthetic. ECharts remains a future option for high-volume visualizations.
- Migration `0012` added `admin_get_overview_chart_series(p_days)`.
- **Chart Data Privacy Model**: The chart RPC is strictly `SECURITY DEFINER` and checks `is_admin()`. It returns only aggregated integers by date. It does **not** return user rows, emails, IP hashes, or event metadata. It deliberately does **not** index `auth.users` to maintain strict auth boundary integrity.
- **Data Series Included**:
  - `usersByDay`
  - `profilesByDay`
  - `securityEventsByDay`
  - `adminAuditByDay`
  - `rateLimitEventsByDay`
- **Zero States**: The UI correctly displays empty zero states when no data is returned or when the series is fully flat.
- **No Fake Data Policy**: All charts display only genuine data generated by the Supabase database. If you don't see chart spikes, it's because there are no recent sign-ups or events.
